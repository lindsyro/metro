# Автоматизированная система информирования пассажиров о причинах задержек городского электрического транспорта

## О проекте

Данное программное обеспечение («Система») предназначено для автоматизации процесса информирования пассажиров о сбоях и задержках в работе городского электрического транспорта (трамваи, троллейбусы, метро). 

Система связывает обращения граждан с реальными диспетчерскими данными, обеспечивая оперативное и прозрачное взаимодействие между транспортной компанией и пассажирами. В качестве интеллектуального ядра для обработки естественного языка используется интеграция с нейросетью GigaChat (LangChain).

### 🎯 Главные цели
* **Сокращение времени реакции** на обращения пассажиров в социальных сетях и мессенджерах.
* **Снижение рутинной нагрузки** на диспетчеров и сотрудников пресс-службы транспортной компании.
* **Повышение лояльности пассажиров** за счёт своевременного и честного информирования о причинах задержек.

## 🚀 Установка и запуск (Инструкция для разработчиков)

### Требования к окружению
Для работы приложения понадобятся:
- **Node.js** (версия 18 и выше)
- **PostgreSQL** (сервер базы данных)

### 1. Клонирование и установка зависимостей
Склонируйте репозиторий и установите необходимые зависимости:
```bash
npm install
```

### 2. Настройка переменных окружения
Создайте файл `.env` в корневой директории проекта и укажите в нём необходимые данные:

```env
# Токен вашего Telegram-бота
BOT_TOKEN=ваш_токен_бота

# ID группы диспетчеров, из которой бот будет автоматически считывать инциденты
DISPATCH_GROUP_ID=-1001234567890

# Авторизационные данные GigaChat API (Credentials)
GIGACHAT_CREDENTIALS=ваш_токен_gigachat

# Строка подключения к PostgreSQL
DATABASE_URL="postgresql://ИМЯ_ПОЛЬЗОВАТЕЛЯ:ПАРОЛЬ@localhost:5432/НАЗВАНИЕ_БД?schema=public"
```

### 3. Настройка базы данных (Prisma)
Система использует Prisma ORM. После настройки подключения к базе в `.env`, выполните миграции:
```bash
# Применение миграций и создание таблиц
npm run db:migrate

# (Опционально) Заполнение базы тестовыми данными
npm run db:seed
```

### 4. Запуск проекта
Для запуска бота в штатном режиме:
```bash
npm start
```
В терминале должно появиться сообщение об успешном подключении к серверам Telegram.

### 5. Запуск тестов
Проект покрыт модульными тестами (Vitest), проверяющими логику работы с базой данных (с использованием моков Prisma).
Чтобы запустить тесты:
```bash
npm test
```
